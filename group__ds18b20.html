Hardware

This section shows, which pin is used to receive the sensor data.

Function GPIOs Remarks
DS18B20_PIN GPIO14 This is data pin to receive the reads.

You could change the ds18b20 params to get another gpio controller.

Basic and Default configuration

For make some changes to ds18_params, you need to change the following structure in ds18b20.h file.

static ds18_params_t therm1_params = {
.pin = GPIO14, /*< Assign the ds18b20 pin */
.out_mode = GPIO_OD_PU, /*< ds18b20 pin mode_out */
.in_mode = GPIO_IN /*< ds18b20 pin mode_in */
};

Flashing

If you want to upload the example to your board, needs to compile first, additional to this you could initialize the terminal and watch the results

make -C examples/esp32-wroom-32/test_ds18b20 flash term

Getting Started:

Follow the next steps:


Build this example in your breadboard



Go to the example folder

cd examples/esp-wroom32/test_ds18b20

Compile your code and flash it

make flash term

Check your code to understand functionalities

First

Define a ds18 sensor

ds18_t sensor

Second

Define ds18 params

ds18_params_t dev ={
.pin = sensor_pin,
.out_mode = sensor_mode_out // OUT_OPRN_DRAIN_PULL UP
.in_mode = sensor_mode_in
}

Last steps

Initialize the device

ds18_init(&dev,&dev_params); // Set the params to your sensor

Check the temperature

ds18_get_temperature(&dev,&var_temperature); // var_temperature has to defined previously as int16_t variable
